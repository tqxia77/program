## 三、统一约定

### 3.1 基础信息
- **Base URL**：`https://your-domain.com/api`
- **Content-Type**：`application/json`
- **认证方式**：`Authorization: Bearer <token>`

### 3.2 响应格式

**成功响应**
```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

**错误响应**
```json
{
  "code": 400,
  "msg": "错误信息",
  "data": null
}
```

**状态码说明**
| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录/Token过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 3.3 分页格式

**请求参数**
```
?page=1&pageSize=20
```

**响应格式**
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "list": [...],
    "total": 100,
    "page": 1,
    "pageSize": 20,
    "totalPages": 5
  }
}
```

### 3.4 文件上传

**请求**
```
POST /api/upload
Content-Type: multipart/form-data

file: [文件]
```

**响应**
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "url": "https://cdn.xxx.com/images/xxx.jpg",
    "filename": "xxx.jpg",
    "size": 12345
  }
}
```

### 3.5 日期格式
- **API返回**：`ISO 8601` 格式（`2024-01-15T10:30:00.000Z`）
- **数据库存储**：UTC 时间

## 五、数据库设计

### 5.1 用户相关

```sql
-- 用户表
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  openid VARCHAR(100) UNIQUE NOT NULL,
  nickname VARCHAR(50) NOT NULL,
  avatar VARCHAR(255),
  phone VARCHAR(20),
  role ENUM('elder', 'child') DEFAULT 'elder',
  bio TEXT,
  notification_settings JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- Token黑名单
CREATE TABLE token_blacklist (
  id VARCHAR(36) PRIMARY KEY,
  token VARCHAR(500) NOT NULL,
  expires_at TIMESTAMP NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 5.2 活动相关

```sql
-- 活动表
CREATE TABLE activities (
  id VARCHAR(36) PRIMARY KEY,
  title VARCHAR(100) NOT NULL,
  category ENUM('health', 'education', 'social', 'volunteer') NOT NULL,
  cover_image VARCHAR(255),
  date DATE NOT NULL,
  time TIME NOT NULL,
  location VARCHAR(100) NOT NULL,
  address VARCHAR(255),
  capacity INT DEFAULT 0,
  description TEXT,
  organizer_id VARCHAR(36) NOT NULL,
  status ENUM('pending', 'ongoing', 'completed', 'cancelled') DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (organizer_id) REFERENCES users(id)
);

-- 活动报名表
CREATE TABLE activity_enrollments (
  id VARCHAR(36) PRIMARY KEY,
  activity_id VARCHAR(36) NOT NULL,
  user_id VARCHAR(36) NOT NULL,
  phone VARCHAR(20),
  remark TEXT,
  enrolled_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (activity_id) REFERENCES activities(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY unique_enrollment (activity_id, user_id)
);
```

### 5.3 帖子相关

```sql
-- 帖子表
CREATE TABLE posts (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  content TEXT NOT NULL,
  images JSON,
  voice_text VARCHAR(500),
  like_count INT DEFAULT 0,
  comment_count INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 评论表
CREATE TABLE comments (
  id VARCHAR(36) PRIMARY KEY,
  post_id VARCHAR(36) NOT NULL,
  user_id VARCHAR(36) NOT NULL,
  content TEXT NOT NULL,
  voice_text VARCHAR(500),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (post_id) REFERENCES posts(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 点赞表
CREATE TABLE post_likes (
  id VARCHAR(36) PRIMARY KEY,
  post_id VARCHAR(36) NOT NULL,
  user_id VARCHAR(36) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_like (post_id, user_id),
  FOREIGN KEY (post_id) REFERENCES posts(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 5.4 社交相关

```sql
-- 关注表
CREATE TABLE follows (
  id VARCHAR(36) PRIMARY KEY,
  follower_id VARCHAR(36) NOT NULL,
  following_id VARCHAR(36) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_follow (follower_id, following_id),
  FOREIGN KEY (follower_id) REFERENCES users(id),
  FOREIGN KEY (following_id) REFERENCES users(id)
);

-- 好友请求表
CREATE TABLE friend_requests (
  id VARCHAR(36) PRIMARY KEY,
  from_user_id VARCHAR(36) NOT NULL,
  to_user_id VARCHAR(36) NOT NULL,
  message TEXT,
  status ENUM('pending', 'accepted', 'rejected') DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (from_user_id) REFERENCES users(id),
  FOREIGN KEY (to_user_id) REFERENCES users(id)
);

-- 好友关系表
CREATE TABLE friends (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  friend_id VARCHAR(36) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_friend (user_id, friend_id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (friend_id) REFERENCES users(id)
);
```

## 六、开发要求

### 6.1 代码规范
- 使用 TypeScript 编写
- 接口参数和响应使用 `camelCase`
- 数据库字段使用 `snake_case`
- 每个接口需要 JSDoc 注释

### 6.2 错误处理
```typescript
// 统一使用 HttpException
throw new HttpException('错误信息', HttpStatus.BAD_REQUEST);

// 或使用自定义异常
throw new BadRequestException('参数错误');
```

### 6.3 认证守卫
```typescript
// 需要登录的接口使用 @Auth()
@Auth()
@Get('profile')
getProfile(@CurrentUser() user: User) {
  return user;
```
