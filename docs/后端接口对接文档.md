## 二、统一约定

### 2.1 基础信息

- **Base URL**：`/api`
- **认证方式**：Header 携带 `Authorization: Bearer <token>`
- **Content-Type**：`application/json`（除文件上传外）

### 2.2 响应格式

**成功响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

**错误响应**：
```json
{
  "code": 400,
  "msg": "参数错误",
  "data": null
}
```

**常见状态码**：
| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录/Token过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 2.3 分页格式

**请求参数**：`page`（页码，默认1）、`pageSize`（每页数量，默认20）

**响应格式**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

### 2.4 文件上传

**接口**：`POST /api/upload`
**Content-Type**：`multipart/form-data`
**字段名**：`file`
**响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "url": "https://xxx.oss-cn-beijing.aliyuncs.com/xxx.jpg"
  }
}
```

**限制**：
- 图片格式：jpg、png、jpeg、webp
- 单文件大小：≤ 5MB
- 最多同时上传 9 张

### 2.5 日期格式

- **日期**：`YYYY-MM-DD`（如：2026-05-08）
- **时间**：`HH:mm`（如：14:30）
- **完整时间**：`YYYY-MM-DD HH:mm`（如：2026-05-08 14:30）

## 三、用户认证模块（后端A负责）

### 3.1 微信登录

```
POST /api/auth/login
```

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | string | 是 | 微信授权码 |
| nickname | string | 是 | 用户昵称 |
| avatar | string | 否 | 头像URL |
| role | string | 是 | 角色：elder（老人）/ child（子女） |

**响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "userInfo": {
      "id": "user_123",
      "nickname": "张三",
      "avatar": "https://xxx.jpg",
      "role": "elder",
      "phone": "138****5678"
    }
  }
}
```

### 3.2 Token 刷新

```
POST /api/auth/refresh
```

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| refreshToken | string | 是 | 刷新Token |

**响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
  }
}
```

### 3.3 登出

```
POST /api/auth/logout
```

**请求头**：`Authorization: Bearer <token>`

**响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": null
}
```

| activityReminder | boolean | 否 | 活动开始提醒 |
| commentReminder | boolean | 否 | 评论提醒 |
| likeReminder | boolean | 否 | 点赞提醒 |

**响应**：

  "code": 200,

  "data": null

### 9.1 用户相关

```sql
-- 用户表
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  nickname VARCHAR(50) NOT NULL,
  avatar VARCHAR(255),
  phone VARCHAR(20),
  role ENUM('elder', 'child', 'community') NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

-- 用户认证表（微信）
CREATE TABLE user_auth (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  wx_openid VARCHAR(100) UNIQUE,
  token VARCHAR(500),
  refresh_token VARCHAR(500),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 9.2 活动相关

```sql
-- 活动表
CREATE TABLE activities (
  id VARCHAR(36) PRIMARY KEY,
  title VARCHAR(100) NOT NULL,
  category VARCHAR(20) NOT NULL,
  image_url VARCHAR(255),
  date DATE NOT NULL,
  time TIME NOT NULL,
  location VARCHAR(200) NOT NULL,
  description TEXT,
  capacity INT NOT NULL,
  enrolled INT DEFAULT 0,
  organizer VARCHAR(100),
  contact_phone VARCHAR(20),

CREATE TABLE activity_signups (

```

### 9.3 帖子相关

```sql
-- 帖子表
CREATE TABLE posts (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  content TEXT NOT NULL,
  voice_text TEXT,
  likes INT DEFAULT 0,
  comments INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

CREATE TABLE post_images (
  id VARCHAR(36) PRIMARY KEY,

  FOREIGN KEY (post_id) REFERENCES posts(id)

CREATE TABLE comments (
  id VARCHAR(36) PRIMARY KEY,
  post_id VARCHAR(36) NOT NULL,
  user_id VARCHAR(36) NOT NULL,
  content TEXT NOT NULL,
  voice_text TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (post_id) REFERENCES posts(id),

CREATE TABLE post_likes (
  id VARCHAR(36) PRIMARY KEY,

```

CREATE TABLE friends (

## 十、开发要求

### 10.1 代码规范

- 使用 TypeScript 编写
- 统一使用 `snake_case` 命名（数据库和API参数）
- 统一使用 `camelCase` 命名（前端JS/TS）
- 接口返回统一使用 `camelCase`

### 10.2 错误处理

- 所有接口需要捕获异常
- 统一错误响应格式
- 日志记录关键操作

### 10.3 安全要求

- Token 验证使用 JWT
- 敏感操作需要验证用户身份
- SQL 注入防护
- 文件上传类型校验

### 10.4 性能要求

- 列表接口必须支持分页
- 图片建议使用 CDN 加速
- 重要接口添加缓存

### 10.5 接口文档

- 每个接口完成后更新本文档标注状态
- 如有变动及时同步给前端同学
